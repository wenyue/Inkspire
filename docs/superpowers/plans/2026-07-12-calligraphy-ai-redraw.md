# Calligraphy AI Redraw Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all 20 calligraphy-related selector images with legible AI redraws, make every non-format option artwork-filled and group-consistent, and add one unique large preview image to each of the 20 popular templates.

**Architecture:** Build five approved high-resolution calligraphy masters first, then use them as reference inputs for the spirit, format, material, entry, hero, and two calligraphy-template assets. Normalize existing non-format painting options with deterministic crops, replace only the inconsistent studio-objects subject, and extend the template model/UI to render 20 unique `960x720` WebPs in responsive large-image cards. Keep generated masters and rejected candidates under ignored `.runtime/selector-review/`, publish only dimension-normalized WebPs, and preserve source-to-style relationships in internal provenance.

**Tech Stack:** Built-in `imagegen`, Node.js 20, Sharp, WebP, Node test runner, Playwright, existing selector asset validator and contact-sheet builder.
