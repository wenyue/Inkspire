# 三个底部页面独立滚动位置设计

## 目标

画案、藏卷和雅匠分别保存自己的纵向滚动位置。用户切换底部标签或刷新页面后，返回任一标签时恢复该标签最后停留的位置，三个页面互不覆盖。

## 已确认边界

- 位置保存范围是当前浏览器标签页的生命周期。
- 刷新页面后继续恢复。
- 关闭当前浏览器标签页后清除，不在以后重新打开网站时恢复旧位置。
- 只记录 `.main-surface` 的纵向滚动位置，不记录作品画廊等局部横向滚动位置。
- 同一底部标签内的步骤、作品详情、调整页和制作页共用该标签的一个滚动位置槽位，不扩展为逐路由记录。

## 状态所有权

滚动位置由 `App` 持有，因为三个底部页面共用 `App` 中的 `.main-surface` 滚动容器，底部标签和路由切换也由 `App` 统一管理。

状态结构按 `studio`、`library`、`experts` 三个 `Tab` 值保存数值位置。内存引用负责切换时的同步读写，`sessionStorage` 负责刷新恢复。读取存储数据时只接受有限的非负数字；缺失、损坏或无效数据回退为 `0`。

## 页面行为

### 标签切换

- 离开当前标签时，读取 `.main-surface.scrollTop` 并保存到当前标签槽位。
- 新标签内容进入共享滚动容器后，在布局绘制前写入目标标签保存的位置，避免先显示旧标签位置再跳动。
- 点击底部标签、浏览器前进或后退造成的标签变化均使用同一保存与恢复逻辑。
- 重复点击当前标签时沿用现有业务行为，不额外改变滚动位置。

### 页面刷新

- 每次保存位置时同步写入 `sessionStorage`。
- `App` 初始化时读取三个位置；刷新后恢复当前路由所属标签的位置。
- 关闭浏览器标签页后由浏览器清除本次会话数据。

### 内容高度变化

- 恢复值超过当前可滚动范围时，不自行计算高度或轮询内容；浏览器将位置限制到当时有效范围。
- 后续内容加载导致页面变长时不再次强制滚动，避免异步请求完成后打断用户已经进行的操作。

## 实现边界

- 不为三个页面创建新的滚动容器。
- 不修改现有标签路由历史、浏览器后退语义或画案流程状态。
- 不引入全局状态库、持久化依赖或逐路由位置表。
- 存储访问必须兼容测试与服务端渲染式环境中 `window` 不可用的情况。

## 验证

- 单元测试确认存储数据的读取、无效值回退和三个标签槽位互相独立。
- 应用测试分别设置画案、藏卷和雅匠的滚动位置，往返切换后确认各自恢复。
- 应用测试模拟重新挂载，确认当前标签从 `sessionStorage` 恢复。
- E2E 在真实共享滚动容器中验证三个标签往返切换不会互相覆盖，并验证刷新后恢复当前标签。
- 运行客户端测试、类型检查、生产构建和相关 Playwright 用例。
