# 画案照片步骤与移动端滚动设计

## 目标

画案问答中，选择图片或拍照不再作为其它步骤下方的附加控件出现，而是成为问答流程的最后一步。用户必须在这个步骤明确选择“相册”、“拍照”或“跳过”，之后才能进入补充要求和生成页。

同时调整画案移动端布局，普通手机视口下问答页不应因为空间足够却仍显示滚动条。只有内容实际超过可用高度时才允许滚动。

## 范围

包含：

- 更新 `Studio` 的前端流程状态。
- 新增照片步骤的进度、返回、上传、跳过和继续行为。
- 移除第一题和普通问答步骤下方的照片条。
- 调整画案问答阶段的移动端高度、间距和滚动样式。
- 更新相关前端测试。

不包含：

- 修改后端上传或生成接口。
- 修改照片处理格式、存储路径或融合图生成逻辑。
- 重设计结果页、藏卷或雅匠页。
- 新增正式订单、登录或云端功能。

## 用户流程

画案流程改为：

```text
作品类型 -> 分支问题 -> 照片步骤 -> 补充要求/生成 -> 结果
```

照片步骤是最后一个必经选择页。进入照片步骤时，页面展示标题“您想要把作品摆在哪里？”以及三个触控操作：

- 相册
- 拍照
- 跳过

用户选择相册或拍照并上传成功后，仍停留在照片步骤，显示轻量照片预览、文件名和“继续”按钮。点击“继续”进入补充要求页。

用户点击“跳过”后，直接进入补充要求页。跳过是显式状态，确保用户不是因为没看到照片入口而遗漏这一步。

## 状态设计

`Studio` 继续保存现有草稿字段：

- `answers`
- `conversationNotes`
- `sourcePhotoPath`
- `selectedPhotoName`
- `recommendedArtworkSize`

新增一个前端状态表示照片步骤是否已经完成：

- `photoStepComplete`

规则：

- 初始为 `false`。
- 上传照片成功后仍为 `false`，直到用户点击“继续”。
- 点击“跳过”后设为 `true`。
- 上传照片后点击“继续”设为 `true`。
- 移除照片只清空照片相关字段，不自动离开照片步骤。
- 重开画案会重置为 `false`。
- 草稿持久化包含该字段，刷新后不会回到错误步骤。

补充要求页只在问答完成且 `photoStepComplete` 为 `true` 时显示。否则问答完成后显示照片步骤。

## 进度与返回

照片步骤计入进度总数。

示例：

- 未选择作品类型时，进度总数为分支题数加作品类型题再加照片步骤。
- 已选择国画或书法后，总数为 `1 + 分支问题数 + 1`。
- 分支最后一题之后，照片页显示最后一步进度，例如 `第 7 / 7 步`。

返回行为：

- 在分支问题中，返回删除上一题答案，保持现有行为。
- 在照片步骤中，返回上一道分支问题，并保留已上传照片。
- 在补充要求页中，返回照片步骤，并保留照片或跳过状态。
- 如果用户从补充要求页返回照片步骤并重新上传，新的照片覆盖旧照片。

## 界面设计

照片步骤复用当前卷轴问答卡容器，不使用弹窗。这样保持流程一致，也避免移动端弹窗高度、键盘和安全区问题。

照片步骤内容：

- 顶部保留进度和返回按钮。
- 主体使用简洁标题。
- 未上传时显示相册、拍照、跳过三个按钮。
- 上传中显示“正在整理照片”状态。
- 上传成功后显示小预览、文件名、移除按钮和“继续”主按钮。
- 上传失败显示现有通用错误，不推进流程。

照片步骤不展示大预览图，避免本来可一屏显示的页面被图片占高。

## 滚动与移动端布局

当前多余滚动主要来自两个因素：

- 照片控件在第一题或完成页之外额外占据高度。
- `main-surface` 使用固定滚动容器，视觉上容易出现不必要的滚动条。

调整策略：

- 普通问答步骤不再渲染照片条或照片预览。
- 问答卡在小屏下压缩外边距、内边距、题图高度和选项高度。
- 选项按钮保持 44px 以上触控高度，但移动端不再为每项保留过高图片框。
- `main-surface` 只在内容超过可用高度时滚动，保留底部 Tab 安全距离。
- 结果页、藏卷和雅匠页仍可自然滚动，不受问答阶段一屏优化限制。

目标视口：

- 390px 宽手机应可正常完成问答，不因照片条显示多余滚动。
- 360px 宽手机允许在文字较长语言下轻微滚动，但不能出现内容重叠或被底部导航遮挡。

## 数据流

照片上传仍调用现有 `uploadPhoto(file)`。

上传成功后：

- 设置 `sourcePhotoPath`。
- 设置 `selectedPhotoName`。
- 创建本地预览 URL。
- 保存 `recommendedArtworkSize`。

生成作品时：

- 如果用户跳过，`source_photo_path` 为空。
- 如果用户上传并继续，`source_photo_path` 使用上传返回路径。
- 现有作品生成成功后自动创建融合图的逻辑保持不变。

## 错误处理

- 用户取消文件选择：不改变状态。
- 上传失败：停留照片步骤，显示现有错误文案。
- 预览图片加载失败：显示现有占位图，不阻塞继续。
- 拍照权限被浏览器拒绝：由文件输入自然失败或取消，不阻塞用户改选相册或跳过。

## 测试

前端测试覆盖：

- 照片控件不再显示在第一题或分支问题下方。
- 完成分支问答后进入照片步骤。
- 未点击相册、拍照或跳过时，不显示生成按钮。
- 点击跳过后进入补充要求页，生成请求不带照片路径。
- 上传照片后显示预览状态和继续按钮。
- 上传后点击继续进入补充要求页，生成请求带照片路径。
- 从补充要求页返回照片步骤，照片状态保留。
- 重开画案会清空照片步骤完成状态和照片数据。
- 进度总数包含照片步骤。

浏览器验证覆盖：

- 390px 手机视口完成问答到照片步骤。
- 普通问答页没有无意义滚动条。
- 照片步骤三个操作可触控。
- 底部 Tab 不遮挡问答、照片步骤或补充要求页内容。
