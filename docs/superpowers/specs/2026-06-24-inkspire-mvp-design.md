# 墨起 MVP 设计规格

## 目标

墨起（Inkspire）是一个本机单用户网页应用，用 Codex 内置图片生成能力生成中国书法和中国画作品。首版面向手机触控使用，重点完成真实生成、精致移动端 UI、收藏夹、专家制作引导、三语界面和浏览器验证。

首版不是营销站。打开应用后直接进入可操作的创作体验。

## 首版范围

### 包含

- React/Vite 前端与 Express 后端。
- 本机单用户运行，后端调用本机 `codex exec --enable image_generation`。
- 主页三 Tab：`画案`、`藏卷`、`雅匠`。
- 简体中文、繁体中文、英文 UI l10n。
- 园林卷轴视觉方向：浅青绿、屏风层次、卷轴式问答、柔和粒子、雅致转场。
- 国画 / 书法二选一分支问答。
- 每个问答问题提供作品预览图和触控选项。
- 问答结束后进入对话补充模式，提供默认建议按钮和自由输入。
- 生成独立作品图。
- 如果用户提供照片，继续生成照片雅化后的融合图。
- 结果页展示作品图和融合图：手机上下拼接，宽屏左右并排。
- 生成成功后自动加入藏卷。
- 结果图下提供制作作品按钮。
- 制作弹窗提供两档服务：专家定制、专家指导。
- 专家、联系方式、价格估算从配置读取。
- 首位专家为用户提供的“广东省吴嘉茵艺术家”。公开资料不足的字段不得编造。
- 图片最终保存为 WebP，以减少容量。
- 启动本地服务并用真实浏览器验证核心体验和动效。

### 不包含

- 登录、多用户、云端同步。
- 支付、正式订单系统、后台管理页。
- 云端部署。
- 让用户编辑完整提示词模板。
- 社交分享发布。

## 技术架构

项目采用 React/Vite + Express：

```text
client/
  React 移动端 UI、l10n、动效、状态展示
server/
  Express API、Codex 任务、图片校验、WebP 转换、文件存储
config/
  应用配置、专家配置、问题配置、提示词模板、三语文案
data/
  本机生成记录、图片、日志、收藏索引
tests/
  API 测试、前端测试、Playwright 浏览器验证
```

前端不直接调用 Codex。所有生成请求进入后端任务队列，由后端创建任务目录、调用 Codex、读取生成结果、校验图片、转换 WebP，并更新记录。

参考 `/home/jinwenhuang/work/OtakuRoomRobot/` 的生成模式，但只复用设计思想，不复制业务结构：

- `codex exec --enable image_generation`
- 任务目录隔离
- `image_generation_end` 事件中的 PNG base64
- 从 `~/.codex/generated_images` 查找新 PNG
- 图片签名和尺寸校验
- 失败诊断与重试
- 全局生成锁

## UI 与交互

### 主导航

三个底部 Tab：

- `画案`：创作工作台。
- `藏卷`：收藏夹。
- `雅匠`：专家介绍。

英文和繁体中文通过 l10n 文件提供翻译。中文 Tab 名保持雅致短词，不使用“工作台 / 收藏夹 / 专家介绍”的直白 UI 名称。

### 画案

画案是首屏。用户可以选择先上传/拍照，也可以直接跳过。

照片入口要求：

- 支持从相册选择。
- 支持调用摄像机拍照。
- 可以跳过。
- 可以先生成作品，后续补图。
- 没有照片时不影响作品生成，只是不生成融合图。

### 卷轴式问答

问答采用屏风/卷轴式单题界面。每题包含：

- 三语题目。
- 一张预览图，用来说明问题含义。
- 2-4 个大触控选项。
- 可选“跳过 / 由墨起决定”默认项。

第一题固定为作品类型：

- 国画
- 书法

后续问题按作品类型分支，不展示无关问题。

国画分支建议问题：

1. 题材：山水、花鸟、人物、器物小品。
2. 气韵：清雅、空灵、繁盛、孤高。
3. 色彩：水墨、浅绛、青绿、设色。
4. 构图：长卷、册页、团扇、竖幅。
5. 题款倾向：无题款、短题、诗意题、印章点缀。
6. 用途：居家点缀、礼赠、头像/屏保、实体制作参考。

书法分支建议问题：

1. 内容方向：自拟短句、诗词意境、祝福语、用户输入文字。
2. 书体：楷、行、草、隶、篆。
3. 气质：端庄、洒脱、苍劲、清润。
4. 版式：横幅、竖幅、斗方、扇面。
5. 墨色：浓墨、淡墨、飞白、朱墨点缀。
6. 用途：居家点缀、礼赠、头像/屏保、实体制作参考。

问题数量保持 5-6 个。后续实现时可在 `config/questions.json` 调整。

### 对话补充模式

问答完成后进入轻对话模式。每轮等待用户输入时，必须提供默认建议按钮，降低输入成本。

默认建议示例：

- 可以开始生成。
- 更清雅一点。
- 留白更多。
- 更适合挂在客厅。
- 更适合送礼。
- 加一点诗意。

用户也可以自由输入。Agent 或后端逻辑判断信息足够后开始生成。

### 结果页

作品图成功后显示作品图，并自动加入藏卷。

如果有用户照片，融合图成功后与作品图一起展示：

- 手机竖屏：作品图在上，融合图在下。
- 宽屏：作品图在左，融合图在右。

融合图要求：

- 对原照片做整体美化，增加雅气。
- 将作品融入原图场景中作为点缀。
- 给作品区域打柔和美光或局部亮度，让其突出显示。
- 不应像粗糙贴图或硬性广告图。

结果页提供：

- 继续生成。
- 补充要求。
- 补图生成融合图。
- 制作作品。

不满意时，用户回到对话补充模式，然后再次生成。新结果保留为新的藏卷记录，不覆盖旧作品，除非后续显式设计删除功能。

## 图片生成流程

生成拆成两个阶段。

### 阶段一：作品图 `artwork`

输入：

- 作品类型。
- 问答答案。
- 对话补充。
- 当前 UI 语言。
- 用户照片是否存在，但作品图不依赖照片。

处理：

1. 后端创建记录 ID 和任务目录。
2. 组装中文艺术提示词。
3. 调用 Codex 图片生成。
4. 接收 PNG 或查找新 PNG。
5. 校验 PNG。
6. 转换为 WebP。
7. 保存 `artwork.webp`。
8. 写入 `record.json` 和 `library.json`。

输出：

- `artwork.webp`
- 作品元数据
- 生成诊断日志

### 阶段二：融合图 `fusion_render`

仅当用户提供照片时执行。

输入：

- 原照片。
- 已生成作品图。
- 问答答案。
- 对话补充。

处理：

1. 将原照片归档到任务目录，最终保存为 `source-photo.webp`。
2. 组装融合图提示词。
3. 调用 Codex 图片生成。
4. 要求照片气质雅化，并把作品自然融入场景。
5. 校验输出并转为 `fusion.webp`。
6. 更新同一条藏卷记录。

输出：

- `fusion.webp`
- 更新后的记录状态
- 生成诊断日志

如果作品图失败，不进入融合阶段。如果融合图失败，保留作品图并允许用户重试融合阶段。

## API 设计

首版 API 以简单轮询为主，不引入 WebSocket。

建议端点：

- `GET /api/health`：检查服务、配置、Codex 可用性、数据目录可写性。
- `GET /api/config/public`：返回公开配置，例如专家服务、价格范围、默认语言。
- `GET /api/library`：读取藏卷列表。
- `GET /api/records/:id`：读取单条记录。
- `GET /api/records/:id/images/:kind`：读取 `artwork`、`fusion` 或 `source-photo`。
- `POST /api/uploads/photo`：上传照片，返回临时照片 ID 或绑定到记录。
- `POST /api/generations`：创建作品生成任务。
- `POST /api/records/:id/fusion`：为已有作品补图生成融合图。
- `POST /api/records/:id/regenerate`：基于补充要求重新生成新作品。
- `GET /api/jobs/:id`：轮询任务状态。
- `POST /api/records/:id/favorite`：更新收藏状态。
- `POST /api/records/:id/production-estimate`：根据服务档位和规格估价。

任务状态：

- `queued`
- `running`
- `succeeded`
- `failed`

任务阶段：

- `artwork`
- `fusion_render`

## 数据模型

本机文件结构：

```text
data/
  library.json
  records/
    <id>/
      record.json
      source-photo.webp
      artwork.webp
      fusion.webp
      codex-events-artwork.jsonl
      codex-output-artwork.txt
      codex-events-fusion.jsonl
      codex-output-fusion.txt
      original/
        artwork.png
        fusion.png
```

`record.json` 字段：

- `id`
- `created_at`
- `updated_at`
- `locale`
- `type`: `painting` 或 `calligraphy`
- `answers`
- `conversation_notes`
- `artwork_prompt`
- `fusion_prompt`
- `source_photo_path`
- `artwork_path`
- `fusion_path`
- `favorite`
- `status`
- `artwork_status`
- `fusion_status`
- `diagnostics`
- `production_interest`

`library.json` 是列表索引，保存每条记录的轻量摘要：

- `id`
- `created_at`
- `type`
- `title`
- `thumbnail_path`
- `has_fusion`
- `favorite`
- `status`

## 配置

### `config/app.json`

包含：

- 应用名。
- 默认语言。
- Codex 命令。
- Codex 模型。
- Codex reasoning effort。
- 生成尺寸。
- WebP 质量。
- 数据目录。
- 任务重试次数。

### `config/experts.json`

包含：

- 专家 ID。
- 姓名。
- 地区。
- 简介。
- 电话。
- 微信号。
- 服务档位。
- 价格估算规则。

首版服务档位：

1. `专家定制`：专家直接创作或深度主导，价格更高。
2. `专家指导`：专家给方向、修改意见或把关，价格更低。

首位专家：

- 姓名：吴嘉茵
- 地区：广东省
- 其他资料：仅写用户确认或配置中提供的内容，不使用未证实公开履历。

### `config/questions.json`

保存国画和书法的问题树。每题包含：

- `id`
- `applies_to`
- `preview_prompt`
- `title` 三语文案
- `options` 三语文案
- `default_option`

预览图可以由静态古风 UI 图、CSS/canvas 绘制图或预先生成的 WebP 承载。首版不在每个问题展示时实时调用 AI 生成预览图，以免问答成本过高、等待过长。问题预览图属于 UI 资产，和用户最终作品生成任务分开。

### `config/i18n/*.json`

文件：

- `zh-Hans.json`
- `zh-Hant.json`
- `en.json`

覆盖：

- Tab
- 按钮
- 问答
- 状态
- 错误提示
- 制作弹窗
- 专家页
- 空状态

生成提示词默认仍以中文艺术语义为主，避免英文或繁中提示词降低书画质量。

### `config/prompts/*.json`

文件：

- `painting.json`
- `calligraphy.json`
- `fusion.json`

保存中文提示词模板与安全边界：

- 避免水印、伪签名、假品牌。
- 避免生成具体在世艺术家风格仿作。
- 允许传统风格术语，如水墨、青绿、飞白、行草、浅绛。
- 书法内容需要尊重用户输入，不生成违法或仇恨内容。

## 专家制作流程

结果页点击“制作作品”打开弹窗。

弹窗内容：

- 作品预览。
- 服务档位：专家定制、专家指导。
- 可选规格：尺寸、复杂度、是否需要融合图作为参考。
- 估算价格。
- 确认按钮。

确认后展示：

- 电话。
- 微信号。
- 提示用户联系后沟通具体事项。

首版不创建真实订单，不收款，不承诺排期。

## 错误处理

启动时检查：

- `codex` 是否可执行。
- 配置 JSON 是否可解析。
- `data/` 是否可写。
- WebP 转换能力是否可用。

生成时：

- 使用全局任务锁，避免并发 Codex 任务争抢生成目录。
- 每个阶段最多重试一次。
- 失败时保存 events、stderr 和最终诊断。
- 返回用户可读错误，不把完整 stderr 直接暴露到 UI。
- 作品阶段失败时记录失败任务。
- 融合阶段失败时保留作品图。
- 照片权限被拒不阻塞作品生成。

图片处理：

- Codex 输出可能是 PNG base64，也可能落到 `generated_images`。
- 后端负责校验 PNG 签名和最小文件大小。
- 后端负责转换 WebP。
- WebP 转换失败时保留原始 PNG 供诊断，但前端不展示未完成结果。

## 视觉与动画要求

整体方向为园林卷轴：

- 主色为浅青绿、纸色、墨色、朱印。
- 避免大面积单一紫色、深蓝或棕橙。
- 不使用营销式大 hero。
- 第一屏必须是可用工作台。
- 粒子效果用于花瓣、墨尘、金粉或纸纤维，不遮挡文字。
- 问答切换有卷轴滑动或屏风翻页感。
- 生成中有墨迹扩散、光晕或粒子流动反馈。
- 结果图出现时有柔和显影动画。
- 所有文字在 390px 手机宽度不重叠、不溢出。

## 测试与验证

### 自动测试

- API：配置读取、健康检查、任务创建、任务锁、失败保存、WebP 输出、藏卷索引。
- 前端：三语切换、问题分支、默认建议按钮、制作弹窗、价格估算。
- 数据：`record.json` 和 `library.json` 读写一致性。
- 图片工具：PNG 识别、WebP 转换、失败路径。

### 浏览器验证

实现完成后必须启动本地服务并使用真实浏览器验证：

- 390px 手机视口可完整操作。
- 主页三个 Tab 可触控。
- 问答流程可完成。
- 默认建议按钮可触发生成。
- 粒子 canvas 或动画层非空且不遮挡 UI。
- 上传/拍照入口存在，拒绝权限时不阻塞。
- 作品图显示正常。
- 有照片时融合图显示正常。
- 手机结果页上下拼接。
- 宽屏结果页左右并排。
- 制作作品弹窗可打开、切换档位、估算价格、关闭。
- 三语切换后主要 UI 文案更新。

### 真实生成验证

至少执行：

1. 一次国画作品生成。
2. 一次书法作品生成。
3. 一次带样例照片的融合图生成。

每次确认：

- 最终展示 WebP。
- 藏卷新增记录。
- 失败诊断文件存在。
- 前端状态和后端记录一致。

## 实施约束

- 不自动提交 Git，除非用户明确要求。
- 设计、计划和文档使用中文。
- 不编造专家公开资料。
- 真实图片生成必须通过后端封装。
- 首版优先完成本机单用户体验，不扩展云端和支付。

## 验收标准

首版完成时应满足：

- 用户可在手机尺寸浏览器中完成从问答到作品生成的完整流程。
- 生成结果为 WebP，并自动保存到藏卷。
- 用户提供照片时，可生成雅化融合图并与作品图拼接展示。
- 用户不满意时，可回到对话补充并继续生成。
- 制作弹窗可按专家定制 / 专家指导估价，并展示配置中的电话和微信。
- 简体中文、繁体中文、英文 UI 可切换。
- 动效、粒子和布局经过真实浏览器验证。
