# 画案八步生成流程设计

## 目标

将画案最后的“补一句想法”界面计为独立步骤。常规国画和书法流程从七步改为八步，让进度与用户实际经历的界面一致。

## 范围

包含：

- 调整画案进度计算，使其能够区分问题、照片、疏密和补充想法界面。
- 将照片界面和无环境照片时的疏密选择界面归入第七步。
- 将补充想法和生成界面计为第八步。
- 保持现有前进、返回、草稿恢复和生成请求行为。
- 更新相关前端单元测试与端到端测试。

不包含：

- 新增或删除问题。
- 修改环境照片、疏密选择或补充想法的字段含义。
- 修改生成接口、提示词或服务端任务流程。
- 重设计画案界面样式。

## 用户流程

常规国画和书法流程为：

```text
作品类型 -> 五个分支问题 -> 照片/疏密设置 -> 补充想法并生成
   1            2-6                7                 8
```

第七步有两条路径：

- 用户提供环境照片：照片界面显示 `第 7 / 8 步`，确认照片后直接进入第八步。
- 用户跳过环境照片：照片界面和随后出现的疏密选择界面都显示 `第 7 / 8 步`；选择疏密后进入第八步。

补充想法和生成界面始终显示 `第 8 / 8 步`。生成按钮继续只出现在该界面。

历代名作路径沿用相同规则：选择名作、照片/疏密设置和补充想法各自按当前路径计数，最后的补充想法界面单独占最后一步。

## 实现设计

采用按当前界面阶段计算进度的方案，不新增持久化状态。

进度计算继续以现有答案和问题配置确定基础步骤数，同时接收当前画案阶段：

- 问题阶段：按当前问题位置显示进度。
- 照片阶段：显示倒数第二步。
- 疏密阶段：仍显示倒数第二步。
- 补充想法阶段：显示最后一步。

总步数在现有基础上增加一，用于独立的补充想法界面。当前的 `photoStepComplete`、`complexityStepComplete` 和路由参数已经足以判断界面阶段，不增加 `notesStepComplete` 等冗余状态。

## 导航与状态

现有导航顺序保持不变：

- 第八步点击“上一步”，无环境照片时返回疏密选择界面。
- 有环境照片且跳过疏密选择时，第八步返回照片界面。
- 浏览器前进和后退继续使用现有 `step=photo`、`step=complexity` 与 `step=notes` 路由。
- 页面刷新和底部标签切换继续从现有草稿与路由恢复，不改变草稿结构。
- 从结果页继续调整时仍使用补充想法界面；它不重新启动完整八步流程。

## 错误处理

本次不引入新的异步操作或错误状态。照片上传失败、生成失败和并发限制继续由现有逻辑处理，失败不会改变当前步骤或进度。

## 测试

前端测试覆盖：

- 常规五题分支在选择作品类型后显示八步总数。
- 照片界面显示 `第 7 / 8 步`。
- 跳过照片后的疏密界面仍显示 `第 7 / 8 步`。
- 补充想法和生成界面显示 `第 8 / 8 步`。
- 提供环境照片并跳过疏密选择时，直接从第七步进入第八步。
- 从第八步返回时，分别落到疏密界面或照片界面。
- 历代名作路径的最终补充想法界面也独立计为最后一步。

端到端测试同步更新可见进度断言，确保实际浏览器流程与单元测试一致。
