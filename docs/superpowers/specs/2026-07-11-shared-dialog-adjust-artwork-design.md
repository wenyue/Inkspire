# 通用 Dialog 与调整作品弹窗设计

## 目标

将“调整作品”从页面内区域改为与“制作作品”一致的模态弹窗，同时移除调整流程中的“当前作品”图片展示。抽取一个不包含业务逻辑的通用 Dialog，让制作作品和调整作品共享模态语义、关闭行为与焦点管理。

## 范围

- 新增通用 Dialog 组件。
- 将 `ProductionDialog` 的通用模态外壳迁移到新组件。
- 将 `AdjustView` 改为使用通用 Dialog。
- 移除 `AdjustView` 中当前作品图片、占位图、图片查看器和相关参数。
- 保持调整提交、建议选项、错误提示和生成流程不变。
- 保持制作作品的尺寸、服务、报价、参考程度及下单逻辑不变。

本次不迁移 `ConfirmDialog`。确认框采用居中布局和独立操作结构，不属于制作作品与调整作品共享的底部弹窗范围。

## 组件设计

新增通用 `Dialog` 组件，负责：

- 渲染固定全屏遮罩与底部弹出的弹窗容器。
- 设置 `role="dialog"`、`aria-modal="true"` 以及标题关联。
- 渲染标题和右上角关闭按钮。
- 在打开时聚焦关闭按钮，并在卸载时恢复原焦点。
- 按 `Escape` 时调用关闭回调。
- 将 `Tab` 和 `Shift+Tab` 焦点循环限制在弹窗内部。
- 在挂载期间隐藏底部导航，卸载时恢复。
- 提供内容区、可选底部操作区和样式修饰入口。

通用组件不读取作品、制作订单或调整意见等业务数据，也不拥有提交逻辑。

## 制作作品弹窗

`ProductionDialog` 保留现有状态和业务逻辑，只移除自身重复的遮罩、标题、关闭按钮、键盘处理、焦点恢复和底部导航隐藏逻辑，改由通用 Dialog 提供。

尺寸子页、成功页、滚动内容区和底部确认按钮继续按现有条件渲染。现有 `production-dialog` 样式作为业务修饰样式保留，布局建立在通用 Dialog 外壳之上。

## 调整作品弹窗

`AdjustView` 使用通用 Dialog，在结果页之上显示。弹窗内容依次为：

1. 标题与右上角关闭按钮。
2. 调整说明。
3. 调整意见输入框及清空按钮。
4. 快捷建议选项。
5. 提交按钮与错误提示。

不再渲染当前作品图片、作品占位图、图片标签或大图查看器。相应的 `record`、`baseLabel`、`artworkLabel` 和翻译函数参数从 `AdjustView` 接口删除；调整生成仍由 `App.tsx` 持有的当前作品数据构造请求。

关闭按钮或 `Escape` 会沿用现有返回逻辑，回到同一作品的结果页。提交成功后继续进入现有生成状态，不改变路由与会话恢复规则。

## 样式

通用样式负责遮罩、底部定位、最大高度、滚动约束、背景、阴影和安全区。制作作品与调整作品使用各自的修饰类控制宽度、内部间距和业务区块布局。

调整弹窗打开时与制作作品弹窗一样隐藏底部导航。删除仅服务于调整作品图片预览的样式；仍被其他页面使用的通用图片样式不改动。

## 可访问性与键盘行为

- 弹窗标题通过 `aria-labelledby` 与对话框关联。
- 关闭按钮使用调用方提供的本地化标签。
- 打开时焦点进入弹窗，关闭后返回触发元素。
- `Escape` 关闭弹窗。
- `Tab` 和 `Shift+Tab` 不会将焦点移出弹窗。
- 提交中禁用提交按钮的现有行为保持不变。

## 测试

- 为通用 Dialog 增加组件测试，验证语义、关闭按钮、`Escape`、焦点循环、初始焦点和焦点恢复。
- 更新 `ProductionDialog` 测试，确认迁移后关键内容与操作仍可用。
- 更新 `AdjustView` 测试，确认其具有模态语义、不显示当前作品、可输入和清空意见、可选择建议、可提交，并支持关闭。
- 更新应用层测试，确认调整路由显示弹窗、关闭后返回结果页，且弹窗打开时底部导航隐藏。
- 运行相关 Vitest 测试、类型检查和 `git diff --check`。

## 完成标准

- “调整作品”以与“制作作品”一致的底部模态窗口呈现。
- 两者共享通用 Dialog 的关闭、键盘和焦点行为。
- 调整弹窗中不出现当前作品或其占位内容。
- 制作作品与调整生成的业务行为无回归。
