# 图片全屏查看器设计

## 目标

用户点击作品图、效果图或调整作品页面的基准作品图时，进入全屏图片查看模式。查看模式需要支持放大、缩小、重置、拖动平移，并在左上角提供返回按钮关闭查看器。

## 范围

- 结果页 `ResultView`：
  - 作品图可点击打开全屏查看器。
  - 效果图可点击打开全屏查看器。
- 调整作品页 `AdjustView`：
  - 基准作品图可点击打开同一套全屏查看器。
- 不改记录 URL、浏览器历史、后端接口或图片加载路径。
- 图片加载失败时继续显示现有错误占位，不打开查看器。

## 推荐方案

新增一个轻量前端组件，例如 `ImageViewer`，由调用方传入图片地址、标签和关闭回调。`ResultView` 和 `AdjustView` 只负责在点击可用图片时设置当前查看图片，查看器内部负责全屏覆盖层、缩放和平移状态。

这样能避免把缩放手势逻辑复制到两个页面，也不需要引入路由级页面。组件边界清晰：页面负责选择图片，查看器负责查看交互。

## 交互细节

- 点击图片打开固定全屏覆盖层。
- 左上角返回按钮关闭覆盖层，关闭后回到原页面，不丢失结果页或调整页状态。
- 底部居中提供缩放控件：
  - 放大
  - 缩小
  - 重置
- 支持鼠标滚轮和触控板缩放。
- 图片放大后支持拖动平移。
- `Escape` 关闭查看器。
- 缩放下限为适屏大小，缩放上限为 `4x`。
- 重置会恢复默认缩放和平移。

## 可访问性

- 覆盖层使用 `role="dialog"` 和 `aria-modal="true"`。
- 返回按钮有明确文本标签，不只依赖图标。
- 缩放按钮有明确 `aria-label`。
- 图片保留原来的语义标签，例如“作品图”“效果图”。
- 打开时焦点进入返回按钮；关闭时页面恢复到原有可操作状态。

## 组件与数据流

- `ResultView.tsx`
  - 为作品图和效果图包一层按钮或可点击容器。
  - 点击时设置本地查看状态：`{ src, alt }`。
  - 传给 `ImageViewer` 渲染。
- `AdjustView.tsx`
  - 基准图可用时设置相同查看状态。
  - 复用 `ImageViewer`。
- `ImageViewer.tsx`
  - 管理 `scale`、`translateX`、`translateY`、拖动状态。
  - 监听键盘关闭和滚轮缩放。
  - 暴露关闭回调，不持有业务数据。
- `styles.css`
  - 增加全屏层、工具按钮、图片舞台、拖动状态样式。
  - 保持现有结果页网格和调整页布局不变。

## 错误处理

- 调用方只在图片存在且未触发加载错误时允许打开查看器。
- 查看器内部如果图片加载失败，显示简短不可用状态并允许返回。
- 缩放和平移状态在每次打开新图片时重置。

## 测试计划

- `client/tests/app.test.tsx`
  - 结果页点击作品图后出现全屏查看器，返回按钮关闭。
  - 结果页存在效果图时点击效果图也能打开。
  - 调整作品页点击基准作品图后出现全屏查看器。
  - 放大、缩小、重置按钮可用，并更新查看器状态。
  - 关闭查看器后原页面仍可继续执行制作或调整等操作。
- 样式或布局检查：
  - 确认全屏覆盖层固定铺满视口。
  - 确认左上角返回按钮在移动端安全区内可点击。

## 验证

- 前端组件和交互变更后运行 `npm test --workspace client`。
- 如果实际实现改变了结果页移动布局、滚动或导航层级，再运行 `npm run e2e`。
