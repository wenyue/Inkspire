# 画案选择图片重设计

## 背景

当前画案选择页的红区主视觉由 `Studio.tsx` 把 `option_preview_images` 中的多张选项图拼成 `preview-montage` 横幅。这个实现让主视觉看起来像素材拼接，而不是一张完整的高质量艺术图。入口页也会把“国画”和“书法”两张选项图并排拼在红区。

蓝区每个选项已经有独立图片，但现有书法选项图偏向粗大、夸张、招牌式视觉，容易产生“江湖风”印象，不适合作为墨起的审美引导。

## 目标

1. 红区每一步只显示一张完整、高质量、有艺术性的主视觉图，不再用选项图拼接。
2. 蓝区每个选项继续显示独立选项图，但入口、国画、书法的选项图全部统一规格和审美质量。
3. 书法选项图重新生成，明确避开江湖体、招牌字、夸张飞白、粗暴大字和廉价印章感。
4. 保持现有选择流程、答案保存、生成 payload 和后端提示词逻辑不变。
5. 移动端红区和蓝区图片都稳定显示，不裁掉关键内容，不造成布局跳动。

## 非目标

- 不让用户进入页面时动态生成选择图片。
- 不改变问题、选项、答案值或后端生成提示词字段。
- 不把图片路径写入用户答案或作品记录。
- 不重做整个问答流程布局。
- 不引入新的远程图片服务或运行时图片接口。

## 已选方案

采用“红区主图和蓝区选项图分开生成、分开配置”的方案。

红区使用 `preview_image`，语义是“这一题的审美方向总览”。它是一张完整主视觉，不偏向某个具体选项，也不把所有选项硬塞到同一张图里。

蓝区使用 `option_preview_images`，语义是“这个选项的可视化提示”。每个选项一张独立缩略图，用户点击选项时仍然只保存选项文本值。

不采用以下方案：

- 主图复用某个选项图：会让用户误以为该选项被默认推荐，且主视觉层级不够独立。
- 运行时动态生成：会拖慢首屏、增加失败状态，并让选择页审美不稳定。

## 数据与资产结构

沿用现有静态资产目录：

```text
client/public/previews/questions/
client/public/previews/options/
```

红区主图放在 `client/public/previews/questions/`，继续通过 `config/questions.json` 的 `preview_image` 引用。

蓝区选项图放在 `client/public/previews/options/`，继续通过 `config/questions.json` 的 `option_preview_images` 引用。

入口问题 `work_type` 在 `client/src/domain.ts` 中定义，也要遵循同一规则：

- `preview_image` 指向一张完整入口主视觉。
- `option_preview_images` 分别指向“国画”和“书法”的高质量选项图。

优先保持现有文件路径不变，用新生成的 WebP 覆盖旧资产。这样可以减少配置、测试和专家样例引用的连锁改动。只有当某张图的语义明显需要拆分或重命名时，才更新对应路径。

## 组件设计

`Studio.tsx` 的问题页结构保持不变：

1. 顶部进度和返回按钮。
2. 红区主视觉。
3. 问题标题。
4. 文本输入或蓝区选项列表。

红区渲染逻辑改为直接显示单张 `preview_image`：

- 复用或保留 `localizedPreviewImage(question, locale)`。
- 红区容器内只渲染一个 `img`。
- 删除 `montagePreviewImages()`。
- 删除 `preview-montage`、`montage-cell`、`montage-tile` 相关渲染依赖。

蓝区选项渲染保持当前逻辑：

- 继续按 `questionOptions(question, locale)` 渲染按钮。
- 继续用 `optionPreviewImage(question, index, locale)` 读取每个选项图。
- 点击选项仍然调用 `answerQuestion(option)`。

红区和蓝区图片都只用于展示和审美引导，不进入生成请求体。

## 资产审美规范

### 红区主图

红区主图应当像一张完整的艺术导览图，而不是多个选项的拼贴。它需要表达问题主题：

- 入口：表达墨起的整体品牌气质，能同时暗示国画与书法，但不是左右拼图。
- 国画主题、设色、气质、构图、细节：表达该问题的选择维度，画面完整、克制、有留白。
- 书法书体、笔势、章法、纸张、墨色：表达书法审美维度，画面要有文人书房、碑帖、宣纸和墨韵气质。

红区主图禁止：

- 多张选项图并排拼贴。
- 廉价国潮素材感。
- 旅游纪念品式山水。
- 过度金色、过度纹理、过度装饰。
- 大字招牌、江湖字、夸张飞白。

### 蓝区选项图

蓝区选项图应当清楚暗示当前选项，但不要变成低质图标。它需要在小尺寸下仍然可读：

- 国画选项图：像高质量画案参考，笔墨层次清楚，构图干净。
- 书法选项图：像文人案头的书法局部或章法样张，克制、典雅、可入居室。
- “由墨起决定”选项：表达综合、留白、由系统判断的气质，不做随机拼贴。

书法蓝区图禁止：

- 粗暴黑色大字占满画面。
- 招牌字、江湖体、商业门头感。
- 不可读的乱草。
- 夸张墨点和廉价印章。
- 模仿具体在世艺术家风格。

## 输出规格

图片继续使用 WebP，并按手机端展示优先的规格落地：

- 红区主图：`1024 x 576`，横向 `16 / 9` 主视觉。该尺寸覆盖手机端高密度屏显示需求，同时避免首屏加载过重。关键主体放在中央安全区域，保证短屏下容器变为 `2 / 1` 或 `3 / 1` 时仍不裁掉核心信息。
- 蓝区选项图：`320 x 240`，统一 `4 / 3` 缩略图。该尺寸远高于当前手机端选项缩略图的实际显示尺寸，能保证清晰度，同时减少静态资源体积。图内保留适当纸边，适配当前 `object-fit: contain` 的缩略容器，不依赖 CSS 额外裁切。

生成原图较大时，落地前压缩为 WebP。单张红区主图控制在 `180 KB` 以内，单张蓝区选项图控制在 `80 KB` 以内；如果质量与体积冲突，优先保证手机首屏加载稳定，再微调压缩质量。

## 验证标准

自动化验证：

- 运行 `npm test --workspace client`。
- 调整或新增组件测试，确认红区直接渲染 `preview_image`。
- 调整或新增样式测试，确认红区不再依赖 `.preview-montage`，且选项图仍保持稳定尺寸。

人工验证：

- 打开选择页入口，确认红区是一张完整主视觉，不是国画和书法的左右拼接。
- 进入国画流程，逐步确认每个红区主图都不是选项拼接。
- 进入书法流程，确认蓝区书法选项图没有江湖风、招牌字和粗暴大字。
- 在常见手机宽度和短屏高度下，确认红区主图和蓝区选项图不重叠、不挤压文字、不裁掉关键内容。

## 风险与处理

图片质量是本次改动的核心风险。只改代码移除拼接不能满足目标，必须同时替换静态资产。

如果部分资产一轮生成不达标，应优先返工资产，不通过 CSS 强行裁切或遮盖。CSS 只负责稳定展示，不负责掩盖图片审美问题。

如果为了保持路径不变而覆盖旧文件，代码 diff 会看不出图片内容变化。完成报告需要明确列出替换的资产目录和验证方式。
