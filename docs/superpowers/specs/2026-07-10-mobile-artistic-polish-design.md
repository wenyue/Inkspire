# 墨起移动端系统性雅化设计

日期：2026-07-10

## 目标

把墨起从带有泛化“中国风”装饰的生成工具，提升为可信、克制、具有严肃书画审美的移动端创作体验。

本设计优先解决五类问题：

1. 小屏主操作不持续可见，用户难以完成任务。
2. 作品形制与固定生成画布矛盾。
3. 书法伪字、伪款、伪印和 AI 仿古陈设图损害专业可信度。
4. 古典名作选择没有真正把原作图片传入生成链路。
5. 分类、提示词与文案混用专业概念，容易诱发满、碎、炫技和江湖气结果。

## 审美原则

- 作品图始终是界面主体，控件退居其后。
- 使用宣纸白、墨色、石青和少量赭石，不使用金边、卷轴纹、木纹、假印章等表面化中国风装饰。
- 不把泼墨、烟云、古色等正当传统语言列为绝对禁项；只禁止无结构的廉价特效、统一滤镜和不符合用户选择的装饰。
- 默认不生成落款和印章。只有未来提供明确的题款内容与位置选择后，才能启用相应能力。
- 所有动效克制、短促，并尊重 `prefers-reduced-motion`。

## 范围与实施顺序

### 第一阶段：移动端任务闭环

适用视口以 `320×568` 和 `390×844` 为强制验收尺寸。

#### 创作补充页

- 主内容保持可滚动。
- “生成”成为位于底部导航上方的持续可见操作区。
- 建议词可滚动或折叠，不能把生成按钮推离首屏。
- 底部操作区不得遮挡输入框、建议词或错误信息。

#### 结果页

- 作品图优先使用可用高度，主操作区固定在底部导航上方。
- “制作作品”为主操作；“调整作品”和“添加环境照片”进入次级操作区。
- 在 `320×568` 下，主操作完整可见且可点击，不与底部导航重叠。

#### 制作弹窗

- 弹窗头部和关闭按钮固定。
- 表单主体独立滚动。
- 最终确认按钮固定在弹窗底部，并覆盖安全区。
- 选中尺寸和服务卡在小屏下压缩信息密度，避免正文反复换行。

#### 经典名作详情与雅匠页

- 名作原图保持完整比例，不为展示 CTA 而裁切。
- “选择此作品”固定在内容滚动区底部。
- 雅匠页先展示身份可信度、代表作和可核验信息，再展示服务说明。

## 第二阶段：生成语义与书法真实性

### 动态画幅

由形制决定生成画布：

| 形制 | 方向 | 建议画布比例 |
| --- | --- | --- |
| 立轴 | 竖向 | `2:3` |
| 横幅 | 横向 | `3:2` |
| 斗方 | 方形 | `1:1` |
| 手卷 | 长横向 | `2:1` |
| 扇面 | 横向扇形构图 | `3:2` |
| 册页 | 方形或近方形 | `1:1` |

- `resolvedOrientation` 与实际传给生成器的 canvas 必须一致。
- 用户补充中的明确方向可以覆盖问题答案。
- 画布比例、尺寸估算和结果展示共用一个形制映射来源。

### 国画 prompt

按以下顺序构建正向约束：

1. 画科与主体关系。
2. 构图经营、开合、疏密和虚实。
3. 笔墨组织，包括勾、皴、擦、点、染的关系。
4. 设色与墨色层次。
5. 纸本或绢本质感。
6. 题款和钤印边界。

负向约束包括：

- 不生成无依据的假印、伪款和不可读装饰文字。
- 不使用统一暖黄仿古滤镜、电影光效、3D 浮雕墨迹或廉价泼墨特效。
- 不把烟云、飞白或墨色变化当作随机装饰；必须服务于构图和气韵。
- 不模仿具体在世艺术家。

### 书法 prompt 与真实性

- 正文必须逐字使用用户输入，不得增删、替换、重排或混入英文、数字和装饰性伪字。
- 默认无落款、无印章、无边框、无室内布景。
- 明确区分书体、气息、形制和纸墨。
- 避免电脑字体贴图、均匀描边、过度飞白、无控制涨墨和江湖体夸张连笔。
- Prompt 只是一道约束，不能作为文字准确性的唯一保证。
- 生成后必须进入文字准确性复核；无法确认正文一致时标记为需要重试，不能直接当作完成作品。

### 复杂度语义

将“简、衡、丰”从质量或细节等级改为疏密与虚实倾向：

- 疏朗：主体集中，虚处充分，信息量低。
- 均衡：主次明确，疏密相间。
- 繁密：层次丰富但仍有气口，不等于填满画面。

## 第三阶段：经典名作参照链路

### 信息架构

- 第一步只选择“国画”或“书法”。
- 国画进入下一步后选择“自由创作”或“取法名作”。
- 经典集合改称“东亚历代绘画”；若产品只想表达中国画，则按地域筛选为“中国历代绘画”。

### 真实视觉参照

- 所选经典作品原图作为只读视觉参考传给 artwork runner。
- 生成器参考构图、笔墨、设色、空间节奏和气韵，不直接复制题跋、印章、人物面貌或局部图像。
- 每幅作品拥有个别化的取法重点，不能继续使用按大类生成的同一模板。

### 数据本地化

- 通过经典数据的拥有者脚本重建 JSON，不手改一百条生成数据。
- 简体中文与繁体中文题名、作者、年代和鉴赏短评必须自然、完整。
- 英文保留馆藏机构提供的正式名称。
- 每条鉴赏短评说明可取之处和不可照搬之处。

## 第四阶段：分类与视觉素材

### 国画选择维度

- “题材”使用易懂但维度一致的入口，例如山水、花卉翎毛、人物、走兽鳞介、清供博古。
- “工笔、写意、白描、没骨”改称“表现方式”，允许未来支持兼容组合；首批至少避免文案暗示它们是严格互斥的画科。
- 气质词不再与设色重复，使用能指导构图和笔墨的描述。

### 书法选择维度

- “章法”改为“形制”。
- “碑拓”改为“碑拓肌理”并明确它是一种视觉取向；若无法控制效果，则从纸墨选项移除。
- 形制与纸墨分别影响画布和材质，不能只拼接成描述性字符串。

### 素材替换

- 撤下所有出现伪汉字、假碑帖和混乱笔画的书法预览。
- 主图不再反复使用暖黄书房、卷轴和文房道具。
- 书法书体以真实、可辨的单字或局部碑帖风格示例为主。
- 纸墨选项展示真实纸色、纤维和墨迹边缘，不使用与选项无关的场景图。
- 雅匠页没有真实头像时可使用克制的印谱式标识，但不能伪造肖像或学历背书。

## 第五阶段：视觉系统与动效

### 藏卷

- 改为两列作品画廊或一列大图卡，缩略图占卡片主要面积。
- “移出”降级为图标操作或卡片菜单，不与查看作品争夺视觉权重。
- 标题、画种和日期在图像下方排列，不覆盖作品主体。

### 经典名作

- 分类横条增加右侧渐隐提示，滚动后自动消失。
- 详情页题名、作者和年代优先显示；完整作品图在可滚动区域中展示。
- 图片使用渐进加载与低清占位，不制造仿古滤镜。

### 看图与生成状态

- 看图背景改为更不透明的深墨色，减少底层页面干扰。
- 真实作品根据可用区域等比放大；E2E 微型占位图不作为尺寸判断依据。
- 生成状态使用缓慢墨晕扩散、阶段文案淡入和图像轻微显影，不使用旋转加载器或飞墨特效。

### 通用动效

- 页面步骤：`140–180ms` 淡入与轻微上移。
- 选项按压：轻微缩放与描边加深。
- 底栏：选中底板平滑移动。
- 弹窗：遮罩淡入与面板轻微上浮。
- 图片：模糊占位到清晰图像的短过渡。
- `prefers-reduced-motion: reduce` 下取消位移、缩放和持续动画。

## 错误处理与可信度

- 书法正文无法验证时，结果页显示“文字需要复核”，并提供重试入口。
- 经典参考图片缺失时不静默退化为文字模板；应提示参考图不可用并回到选择页。
- 雅匠身份或服务资料未核验时不展示权威性徽章或确定性资历。
- 动态画布不支持某一比例时明确回退规则，不让生成器同时接收互相矛盾的方向和画布。

## 测试与验收

### 自动化

- Client 单元测试覆盖移动端固定操作区、底栏安全区、弹窗结构和术语。
- Server 单元测试覆盖形制到 canvas 的映射、prompt 正负约束和经典参考图片传递。
- E2E 覆盖 `320×568`、`390×844` 的国画、书法、经典名作、结果、调整、制作和藏卷流程。
- 任何页面不得产生全局横向溢出。
- 主操作在两种验收尺寸下必须完整可见或固定可达。

### 艺术验收

- 分别生成国画竖幅、横幅、斗方和书法样例。
- 逐张检查正文、伪字、落款、印章、画幅、构图、笔墨、设色和材质。
- 使用真实生成样例验收，不能只依赖 E2E 渐变占位图。
- 出现伪汉字、假印、电脑字体感、无结构泼墨、统一暖黄滤镜或电影光效时判定不合格。

## 分批交付

首批实施包括：移动端任务闭环、动态画幅、书法真实性约束、经典参考图片传递、核心术语修正。

第二批包括：经典数据重建、分类体系调整、预览素材替换、藏卷画廊和雅匠可信度设计。

第三批包括：克制动效、真实生成艺术验收以及根据新一轮移动端审查继续修订。
