# 选择器统一视觉系统设计

## 背景

画案问答目前共有 45 个可选择项，其中 40 项带有缩略图，5 个书体选项采用文字与取法说明。现有图片并非只有个别裁剪错误，而是同时存在图集切割残片、主体截断、缩略后不可辨认、选项语义混淆、形制比例失真和视觉来源不统一等系统性问题。

顶部主视觉也把多个题材或风格直接混入同一幅山水，虽然覆盖了多个选项，却缺乏空间秩序，产生素材清单式的杂糅感。

本设计建立一套统一的空间叙事与选项图规范。内容合格的旧图只保留主题参考，不直接与新素材混用；全部可视选项都要经过统一重制或统一校正。

## 目标

1. 审查并统一全部 45 个选择项，不以已发现的“人物”和“走兽游鱼”为局部修补边界。
2. 顶部主视觉形成从建筑空间到纸面细节的连续镜头语言。
3. 不同风格落在独立的画框、画稿或册页上，不再混画成一幅综合山水。
4. 所有选项图使用统一尺寸、纸色、光线、主体比例和安全边距。
5. 选项图在常规手机和短屏紧凑布局中仍能准确辨认。
6. 书法视觉坚持可靠碑帖和法书来源，不生成伪文字或江湖体样张。
7. 国画问题顺序与视觉尺度保持一致，并符合先定题材与形制、再定笔墨设色和气质的创作逻辑。
8. 下方选项图的界面显示尺寸在现有基础上统一放大约 30%，提升手机端辨识度。

## 非目标

- 不在运行时动态生成选择图片。
- 不改变选项的答案值、生成请求字段或后端提示词含义。
- 不把图片路径写入用户作品记录。
- 不通过 CSS 放大或遮挡来掩盖原始素材缺陷。
- 不自动刷新经典名作生成资产。
- 不为视觉统一引入新的远程图片服务。

## 总体视觉概念

### 连续的观看路径

顶部主视觉表现一次逐渐走近作品的观看过程：

```text
东方艺术馆全景
→ 主题展墙
→ 形制陈列间
→ 鉴赏画案
→ 设色样本册
→ 册页近观
```

空间从建筑尺度收缩到纸面尺度，镜头持续向内推进，不在最后重新跳回大场景。

### 统一的艺术空间

所有顶部主视觉属于同一套空间世界：

- 建筑与家具使用克制的东方木构语言，不做影视古装布景。
- 主色为宣纸暖白、旧木褐、墨青灰和少量矿物色。
- 使用柔和自然侧光，避免商业棚拍高光和重度金色装饰。
- 画框、立轴、画案和画册具有一致的材质与年代感。
- 镜头透视、色温和颗粒质感连续，形成同一地点中的观看体验。

### 多风格的陈列方法

多个选项不得直接融合为一幅作品。每种题材、技法、设色或气质必须位于独立载体：

- 画廊阶段使用独立画框、立轴或屏风。
- 画案阶段使用独立画稿、装裱样片或纸本。
- 画册阶段使用连续但边界清楚的册页或折页。

主视觉使用三级层级，而不是让所有元素平均抢夺注意力：

- 约 60% 的视觉权重属于当前步骤的核心作品。
- 约 25% 属于两个次要作品。
- 约 15% 用于其余选项和空间线索。

下方选项图负责逐项准确说明，顶部主视觉负责建立审美维度和观看氛围，因此顶部不要求所有选项等大陈列。

## 国画流程与主视觉映射

国画选择顺序调整为：

```text
题材 → 形制 → 笔墨 → 设色 → 气质
```

形制决定构图边界，应在笔墨和设色之前确认。

| 步骤 | 主视觉场景 | 画面任务 |
| --- | --- | --- |
| 创作方向 | 东方艺术馆全景 | 以国画与书法两个清楚展区建立品牌世界 |
| 国画题材 | 主题展墙 | 山水、花鸟、人物、走兽鳞介、清供雅物分别装裱陈列 |
| 国画形制 | 形制陈列间 | 真实展示横幅、立轴、斗方、手卷、扇面的物理比例和观看方式 |
| 国画笔墨 | 鉴赏画案 | 工笔、写意、白描、没骨分别置于独立画稿，比较用笔与造型方法 |
| 国画设色 | 设色样本册 | 水墨、青绿、浅绛、重彩保持相同题材和构图，只比较颜色体系 |
| 国画气质 | 册页近观 | 以连续折页表现清雅、空灵、雄浑、古拙、明丽的空间节奏差异 |

## 选项图统一规范

### 生成与取材

- 每个选项独立生成、独立取材或独立构图。
- 禁止先生成多格大图再自动切割选项图。
- 不从已损坏的 320×240 图片继续裁切。
- 工作母版至少为 960×720，人工确认后输出 320×240 WebP。
- 内容合格的旧图可以作为主题参考，但必须进入同一套色温、纸色、边距和缩放流程。

### 构图

- 统一使用 4:3 画布。
- 主体包围框不得接触画面边缘。
- 人物、动物、器物、扇面和完整形制四周至少保留 12% 安全区。
- 单一主体通常占画面宽高的 65% 至 78%，避免过小或贴边。
- 不用相邻残片、装饰边框或虚假的卷轴道具填补空白。
- 同一维度保持题材和基本构图尽量一致，只改变正在比较的变量。

### 色彩与材质

- 统一暖宣纸底色和自然墨色，不使用纯白数字背景。
- 国画设色允许矿物青绿、浅绛和重彩差异，但饱和度保持典雅。
- 书法图片保留真实纸张、墨色和装裱信息，不添加廉价印章或仿古滤镜。
- 所有图片使用一致的色彩空间和 WebP 压缩策略。

### 缩略验收

每张图必须同时检查：

1. 原始输出 320×240。
2. 常规手机实际显示约 99×78，较现有 76×60 放大约 30%。
3. 短屏紧凑显示约 73×57，较现有 56×44 放大约 30%。

缩小后仍需满足：主体完整、选项含义可辨、相邻选项差异明显、无残片白边、无断肢截头。

### 界面占位

- 放大的是选项图在按钮中的实际显示区域，源文件仍保持 320×240 WebP，不进行低清晰度插值放大。
- 常规手机选项图目标宽度为 99px、高度为 78px；允许为适配整数像素在 98 至 100px、77 至 79px 范围微调。
- 短屏紧凑布局目标宽度为 73px、高度为 57px；允许在 72 至 74px、56 至 58px 范围微调。
- 选项按钮的最小高度和网格第一列同步增加，不能让放大的图片覆盖文字、挤压点击区或造成横向溢出。
- 320px 宽手机上，图片、间距和选项名称必须在单行网格内完整容纳；较长英文标签允许自然换行。
- 图片仍完整显示，不通过 `object-fit: cover` 二次裁掉主体。

## 国画选项专项要求

### 题材

- 人物必须显示完整头部、躯干和足部，避免只剩衣袍下半段。
- 走兽游鱼采用同一溪岸空间组织，岸上走兽与水中游鱼通过水岸相连，不做左右拼图。
- 文房雅物突出笔、砚、香具或清供器物，避免太湖石压倒主题。
- 山水和花鸟保留清楚焦点，不使用细节密度过高的全景。

### 形制

- 横幅、立轴、斗方、手卷、扇面分别独立制作，不复用同一方形作品套入不同边框。
- 形制的长宽比例、装裱结构、卷轴方向和观看方式必须真实。
- 缩略图可以使用空间陈列，但形制轮廓必须成为第一识别信息。

### 笔墨

- 工笔突出严谨线描与层层设色，不能只用一幅普通人物场景代表。
- 写意突出概括笔墨和墨色变化。
- 白描使用少量完整主体，避免大量细小人物在缩略图中变成噪点。
- 没骨突出无墨线的色墨造型，并清除无意义纸边。

### 设色

- 水墨、青绿、浅绛、重彩尽量使用同一题材和近似构图进行比较。
- 重彩必须与青绿形成明确信息差异，但不使用俗艳高饱和。

### 气质

- 清雅强调疏朗、克制和清润。
- 空灵强调大面积留白、远近消隐和轻薄空气感。
- 雄浑强调大尺度山体、强结构和厚重墨层，不能使用普通小溪场景。
- 古拙强调简朴造型、沉着线质和不媚俗的节奏，不能直接等同于写意游鱼。
- 明丽强调清澈色彩与明净光感，不能继续呈现灰粉色调。

## 书法选项专项要求

### 书体

楷书、行书、草书、隶书、篆书继续使用可靠碑帖或法书取法说明。若加入图片：

- 只使用经过核验的历史碑帖或法书局部。
- 每张图显示完整两至三列章法，不截成孤立大字。
- 图片与取法作品名称同时出现。
- 不使用 AI 生成的文字或模仿在世书家的字样。

### 气息

书法气息不能用不同书体代替：

- 古拙不等于篆书。
- 温润不等于草书。
- 雄强不等于把字放大并截断。

端庄、俊逸、雄强、古拙、温润应尽量在相同或相近书体基准下比较行距、结体、用笔、墨色和章法节奏。素材同样来自可靠书迹，并保留完整局部。

### 形制

- 立轴、横幅、斗方、手卷、册页使用真实装裱比例。
- 手卷必须表现横向展开的连续阅读，不把方形作品放进卷轴模型。
- 横幅必须在缩略图中保持明确横向比例。
- 册页需要放大到手机端可以识别开合结构。

### 纸墨

- 素宣突出纸张纤维、渗化和自然白度，墨痕不应压倒纸面。
- 仿古突出温润旧色，不做脏污滤镜。
- 洒金突出克制金屑与纸色关系，不做闪亮商业金箔。
- 碑拓肌理需要保留拓片边界、字口或纹样线索，不能缩成纯黑噪点。

## 已确认的主要问题

必须整体替换的典型问题包括：

- 人物头部缺失。
- 走兽游鱼题材错误且带相邻图残片。
- 历代名作入口错误复用题材综合图。
- 雄浑和古拙气质语义错误。
- 国画五种形制均有图集残片或形制混淆。
- 多个书法气息选项用书体替代气息，并存在截字。
- 书法横幅、手卷和碑拓肌理缩略后失真。

其余内容合格图片也必须经过统一视觉处理，不能原样混入新资产。

## 数据与组件边界

- 继续使用现有 `preview_image` 表达顶部主视觉。
- 继续使用 `option_preview_images` 表达下方选项图。
- 图片路径仍由 `config/questions.json` 和入口问题配置拥有。
- 组件继续按选项索引读取图片，不把视觉资产写入答案。
- 只有当问题顺序调整时，更新共享配置顺序以及对应进度和流程测试。
- 不为单张图片增加运行时裁剪逻辑；主体位置问题在资产阶段解决。

## 验收方式

### 自动检查

- 所有已配置图片路径存在。
- 所有选项图尺寸和格式符合要求。
- 每个选项与图片索引一一对应。
- 调整后的国画问题顺序在三种语言中保持一致。
- 现有客户端流程、进度、刷新恢复和移动端布局测试通过。

### 人工审美检查

- 生成一张包含全部 40 个视觉选项的带名称联系表。
- 分别以 320×240、99×78 和 73×57 输出联系表。
- 检查主体完整性、语义、选项间区分度、纸边、残片、色温和风格统一性。
- 检查顶部主视觉的空间尺度是否连续收缩。
- 检查书法素材是否可靠、完整且无伪文字。
- 在 320px 和 390px 手机宽度完整走一遍国画与书法流程。
- 检查放大 30% 后的选项图没有挤压文字、遮挡底部操作区或产生横向溢出。

任何自动检查通过但人工审美检查不合格的素材，都不能进入最终版本。

## 实施分批建议

1. 先建立统一空间、色彩、尺寸和安全区基准，并制作一张顶部主视觉与一组国画题材选项作为样板。
2. 样板通过后，批量完成国画题材、形制、笔墨、设色和气质。
3. 再处理书法气息、形制和纸墨；书体素材单独执行可靠来源审核。
4. 最后生成全量联系表，逐项返工不合格素材，再接入配置和组件验证。

样板未通过前不批量生成，以免统一放大错误的空间风格、色温或构图规则。
