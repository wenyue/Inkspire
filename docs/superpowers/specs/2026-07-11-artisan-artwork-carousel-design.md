# 雅匠页无价格文案与单行作品预览设计

## 目标

雅匠页只介绍专家身份、代表作品与服务内容，不展示或暗示价格信息；代表作品改为更精致的响应式单行画廊，在紧凑手机布局下一屏展示两件作品，宽屏按可用空间展示更多作品。

## 已确认边界

- “不提价格”仅适用于雅匠页。
- 雅匠页删除价格、金额、费用、报价、估算等相关文案。
- 制作作品弹窗继续使用现有 `priceEstimate` 数据并展示估价，不改变制作流程。
- 专家配置中的 `priceEstimate` 字段继续保留，雅匠组件不得渲染这些数据。
- 本次不改变吴嘉茵的专家身份、资料、作品图片或作品顺序。

## 雅匠页文案

- 删除专家姓名旁的“价格按需求评估”提示。
- 凭证标签继续展示，但其无障碍名称改用与资历相关的专用文案，不再复用价格提示。
- 专家定制与专家指导的介绍只说明吴嘉茵的参与方式，不提平台金额或估价。
- 页尾边界提示只说明服务范围、修改轮次与交付时间以承接确认单为准，不提费用或报价。
- 简体中文、繁体中文和英文保持同一语义边界。

## 作品预览布局

作品列表使用原生横向滚动的单行弹性布局：

- `flex-wrap: nowrap` 保证任何视口下都只有一行。
- 容器自身设置横向滚动，页面主体不得出现横向溢出。
- 紧凑布局下，每张作品卡宽度为可用宽度减去间距后的一半，因此首屏恰好显示两件完整作品。
- 宽屏下作品卡采用受限的响应式宽度，使三件现有作品可同时显示；未来作品超过可用宽度时继续横向滚动。
- 启用滚动吸附，使触摸或触控板滑动停止时作品自然对齐。
- 保持当前作品顺序，并移除只显示前三件的代码限制，让未来新增作品也能进入同一行画廊。

## 视觉处理

- 作品卡保持竖向比例与 `object-fit: contain`，不得裁切正文、题款或印章。
- 画框与页面宣纸色自然融合，使用细边框、圆角和轻阴影统一不同底色与比例的作品；图片加载表面本身不添加图案或底色。
- 为滚动容器预留少量底部空间并使用细滚动条，既提示可以横向浏览，也不抢夺作品主体注意力。
- 不增加 JavaScript 轮播状态、自动播放或分页圆点；交互依赖浏览器原生滚动，以保证触摸、触控板和键盘操作稳定。

## 无障碍与响应式要求

- 作品容器使用列表语义和代表作品标题作为无障碍名称，每张作品使用列表项语义。
- 图片替代文本继续使用本地化的“代表作品 N”。
- `320×568` 与 `390×844` 下首屏显示两件作品，第三件需要横向滑动查看。
- 宽屏下同一行显示三件现有作品；若未来作品更多，只允许画廊自身横向滚动。

## 验证

- 组件测试确认雅匠页不再渲染价格提示，并确认所有代表作品都进入带列表语义的画廊。
- 应用测试确认雅匠页的三语服务边界不含价格语义，同时制作弹窗的估价行为保持不变。
- E2E 在 `320×568` 和 `390×844` 验证首屏两件、第三件可横向滚动、作品共处一行且页面无横向溢出。
- E2E 在宽屏验证三件现有作品同处一行并同时落入画廊可视区域。
- 运行客户端测试、类型检查、生产构建和相关 Playwright 用例。
