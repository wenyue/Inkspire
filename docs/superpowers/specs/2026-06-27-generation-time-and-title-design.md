# 生成耗时与作品命名设计

## 背景

当前生成中页面统一显示“通常约 30 秒”，但实际用户动作分为两类：

- 只生成一张图：作品图，或已有作品后单独生成效果图。
- 连续生成两张图：画案中带环境图生成，先产出作品图，再自动产出摆放效果图。

当前作品标题由服务端 `server/src/jobs.js` 的 `titleFromRequest()` 生成。书法作品已经使用用户填写的正文作为标题，国画作品主要使用主题答案，例如“山水”“花鸟”，标题不够像作品名。

## 目标

1. 只生成作品图或只生成效果图时，生成中页面显示约 30 秒。
2. 一次用户动作会连续生成作品图和效果图时，生成中页面显示约 50 秒。
3. 书法作品标题保存完整正文；展示位置只显示一行，空间不足时用省略号。
4. 国画作品生成雅致、贴合主题的确定性标题，避免只显示“山水”等分类词。

## 非目标

- 不引入模型二次命名或新的异步命名请求。
- 不新增服务端字段保存“展示标题”。
- 不改变生成队列、并发限制、自动效果图生成流程。
- 不修改真实图片生成 prompt 的语义，除非标题生成测试证明必须共享数据。

## 设计

### 生成耗时文案

在前端生成会话中判断当前任务是否代表“两张图”的用户动作：

- `operation === "create"` 且 payload 中有 `source_photo_path`，代表画案带环境图生成，最终会先生成作品图再自动生成效果图，显示约 50 秒。
- 其他生成任务显示约 30 秒，包括：
  - 不带环境图的作品图生成。
  - 已有作品后单独补效果图。
  - 调整作品。

`GeneratingView` 不再读取单一 `generationLoading.estimate`，而是根据 props 选择 `generationLoading.estimate.single` 或 `generationLoading.estimate.double`。中文、繁体、英文三套文案都同步更新。

### 书法标题

服务端继续用 `answers.text` 作为书法作品 `title`，保留完整正文，避免在数据层按固定字数截断。

前端在标题展示位置保证单行显示，并由 CSS 根据真实容器宽度省略：

- 藏卷卡片标题。
- 作品结果页中可能展示标题的区域；如果当前结果页没有展示作品标题，则不新增标题。
- Studio 活跃任务摘要中使用 job title 的位置。

这些位置使用 `white-space: nowrap; overflow: hidden; text-overflow: ellipsis; min-width: 0;` 等现有样式模式。

### 国画标题

服务端新增一个本地、确定性的标题生成 helper，用用户答案组合标题：

- 主题优先来自 `painting_subject`，忽略“由墨起决定”等默认兜底答案。
- 气质来自 `painting_mood`，设色来自 `painting_palette`，构图或细节可作为辅助。
- 常见主题映射到雅致标题词库，例如山水、花鸟、人物分别有不同标题候选。
- 用 record 输入中的答案稳定选取候选，确保同样答案生成同样标题，便于测试和复现。
- 信息不足时回退到雅致默认标题，而不是“中国画作品”。

示例方向：

- 山水 + 清雅：可生成类似“云岫清音”“溪山入梦”的标题。
- 花鸟 + 温润：可生成类似“花影和鸣”“春枝含韵”的标题。
- 人物 + 雄浑：可生成类似“高士临风”“古意风骨”的标题。

标题生成只影响 record/job 的 `title`，不影响 prompt 内容和生成流程。

## 数据流

1. 用户在画案提交生成。
2. 前端把生成参数保存到 `GenerationSession.payload`。
3. `GeneratingView` 根据会话 payload 判断显示 30 秒或 50 秒。
4. 服务端创建 record/job 时调用 `titleFromRequest()`。
5. `titleFromRequest()`：
   - 书法返回完整 `answers.text`。
   - 国画返回雅致标题。
6. 藏卷和活跃任务摘要按一行省略展示标题。

## 错误处理与回退

- 书法正文为空时仍回退为“书法作品”。
- 国画答案为空、默认项过多或主题无法识别时，使用默认雅致标题。
- 前端无法识别 payload 时按单图 30 秒显示，避免夸大等待时间。

## 验证

服务端：

- 增加或扩展 `server/tests/jobs.test.js`：
  - 书法长正文完整入库。
  - 国画 `painting_subject: "山水"` 不再只用“山水”作标题。
  - 同一组国画答案生成稳定标题。

前端：

- 增加或扩展 `client/tests/generatingView.test.tsx`：
  - 单图显示约 30 秒。
  - 带环境图的创建会话显示约 50 秒。
- 增加或扩展藏卷/样式测试：
  - 标题容器具备单行省略样式。

运行：

- `npm test --workspace server`
- `npm test --workspace client`

如修改只落在这些范围，暂不运行真实 Codex 生成验证。

## 自检

- 没有新增数据字段，避免迁移和兼容成本。
- 没有固定截断书法标题字数，满足“看位置够不够，只能显示一行”的要求。
- 国画命名在服务端完成，藏卷、结果页、任务摘要能拿到同一标题。
- 生成耗时判断沿用现有自动效果图流程，不改变队列行为。
