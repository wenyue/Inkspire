# App 化滚动条可见性设计

## 目标

降低三个底部主页面的网页感，同时保证雅匠作品横向内容仍然容易被发现。

## 设计

- 隐藏 `.main-surface` 的纵向滚动条，但保留触摸、触控板、鼠标滚轮、滚动条拖动以外的键盘滚动能力，以及现有的独立滚动位置恢复。
- Firefox 使用 `scrollbar-width: none`；Chromium、Safari 和 WebKit 使用 `::-webkit-scrollbar` 隐藏主页面滚动条。
- 不修改雅匠作品区的横向滚动行为。该区域继续使用现有的 5px 主题化细滚动条；只有内容溢出时浏览器才会提供可滚动提示。
- 不隐藏其他局部横向滚动区域，避免降低内容可发现性。

## 验收标准

1. 三个主页面均不显示纵向滚动条。
2. 页面仍可通过触摸、触控板、鼠标滚轮和键盘滚动。
3. 三个页面的独立滚动位置与刷新恢复继续工作。
4. 雅匠作品超过可见数量时仍显示细横向滚动条并可左右滚动。
5. Chromium、Firefox 和 WebKit 的端到端测试通过。

## 测试策略

- 先添加样式回归测试，验证主滚动容器隐藏滚动条而作品横向滚动条仍保留。
- 运行客户端相关测试与生产构建。
- 运行现有跨浏览器滚动位置与雅匠作品端到端场景。
